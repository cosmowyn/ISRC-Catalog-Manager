# ISRC Catalog Manager 6.0.2

[![Release](https://img.shields.io/github/v/tag/cosmowyn/ISRC-Catalog-Manager?label=release)](https://github.com/cosmowyn/ISRC-Catalog-Manager/releases/tag/v6.0.2)

Version: 6.0.2
Date: 2026-06-03
Type of update: Patch

## Fixes
- Fixed a post-open failure path for encrypted profiles created from the new profile workflow.
- The current-owner bootstrap prompt now returns cleanly when dismissed or left without a selection, keeping the encrypted profile open so the owner can be set later in Party Manager.
- Hardened the catalog-number registry selector so transient registry lookup failures during profile activation clear suggestions/status instead of failing the profile open sequence.

## Update Notes
- This release supersedes v6.0.1 for automatic updates.
- New encrypted profiles created in v6.0.1 remain valid. Reopen them with the password chosen during profile creation.
- If the current-owner prompt appears after opening a profile, it can be dismissed safely; use Party Manager later to set the Current Owner Party.

## Validation Focus
- Reproduced encrypted profile creation/open locally with a rejected owner prompt and verified the profile stayed open.
- Added controller, app-shell, and code-registry widget regression coverage for the post-open failure paths.
- Re-ran focused database security/session tests to confirm encrypted profile creation and reopen behavior remain intact.
