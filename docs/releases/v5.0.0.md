# ISRC Catalog Manager 5.0.0

[![Release](https://img.shields.io/github/v/tag/cosmowyn/ISRC-Catalog-Manager?label=release)](https://github.com/cosmowyn/ISRC-Catalog-Manager/releases/tag/v5.0.0)

Version: 5.0.0
Date: 2026-06-01
Type of update: Major

## Highlights
- Added a production-grade crash and manual bug reporting system with startup-based crash detection, explicit user consent, sanitised report previews, local pending-report storage, local rate limiting, and secure GitHub issue submission through a server-side report proxy.
- Added structured GitHub issue forms and repository reporting labels for bug and crash reports, plus a label configuration script that never stores credentials locally.
- Added a reference GitHub App report proxy that keeps GitHub write credentials server-side, validates report schema, enforces accepted labels and title prefixes, rate-limits submissions, applies a second sanitisation pass, and creates issues with a short-lived installation token.
- Expanded the forensic and authenticity workflows for SoundCloud-oriented evidence handling, degraded derivative verification, more truthful confidence reporting, and broader audio-format handling.
- Promoted UI process qualification from deterministic harness shortcuts toward user-led workflows with screenshot capture, baseline comparison, generated artifact checks, help coverage validation, and traceability artifacts.
- Added invoice and accounting workspace infrastructure with UI-led qualification coverage for invoices, statements, payouts, credit notes, reports, and ledger-backed workflow evidence.

## Crash and Bug Reporting
- New `isrc_manager.reporting` package separates crash detection, diagnostics collection, sanitisation, payload modelling, rate limiting, pending storage, proxy submission, controller wiring, and PySide6 dialogs.
- Startup now opens a reporting session marker after logging is configured. Clean shutdown closes the marker. An unclean previous marker triggers a next-start prompt before report generation.
- Crash prompts explain what diagnostics may be included and clearly state that no report is sent without explicit consent.
- Manual reports are available from `Help > Report a Bug...` and collect summary, description, steps to reproduce, expected behaviour, actual behaviour, optional sanitised logs, and optional system details.
- Report previews show the exact Markdown payload before submission and allow copying or cancellation.
- Sanitisation redacts secrets, passwords, API keys, bearer tokens, GitHub tokens, OAuth tokens, connection strings, private keys, email addresses, phone numbers, local usernames, home paths, binary data, and obvious raw database row content.
- Reports are sanitised both during construction and again at the submission boundary before proxy submission or local pending storage.
- If no secure proxy is configured, reports are saved locally under the app data `reports/pending` folder instead of attempting direct GitHub writes.

## Repository and Proxy Configuration
- Added `.github/ISSUE_TEMPLATE/bug_report.yml` and `.github/ISSUE_TEMPLATE/crash_report.yml` so public reports use structured forms.
- Disabled blank GitHub issues in `.github/ISSUE_TEMPLATE/config.yml` and linked reporters to the reporting privacy model.
- Added `scripts/configure_github_reporting.py` to create or update the required `bug`, `user-report`, and `crash-report` labels using `GH_TOKEN` or `GITHUB_TOKEN` from the environment only.
- Added `docs/reporting.md` with the security model, GitHub App setup, proxy deployment variables, privacy behavior, rate limits, and known limitations.
- The reference proxy requires a GitHub App installed only on the target repository with Metadata read-only and Issues read/write permissions. It does not require Contents, Pull requests, Actions, Administration, Secrets, or Workflow permissions.

## UI Qualification and Help Governance
- Added automated UI PQ coverage for visual qualification, screenshot capture, baseline comparison, generated document/report verification, PDF comparison, dialog verification, theme verification, and help documentation validation.
- Added help documentation requirements to `AGENTS.md`: user-visible feature changes must update the in-app Help manual and be governed by QA/PQ tests.
- Added a help screenshot refresh workflow so committed help screenshots can be regenerated from current UI surfaces.
- Expanded help content to explain crash and bug reporting privacy behavior, report previews, pending local reports, and support workflows.
- Updated traceability evidence, UI PQ summaries, and validation artifacts for the release.

## SoundCloud, Authenticity, and Forensics
- Adjusted authenticity verification language and thresholds so high-confidence degraded derivatives are reported honestly as likely matches instead of being rejected solely by a strict verified threshold.
- Added conversion support for non-WAV authenticity checks so lossy rips and streamed/downloaded derivatives can be tested directly after safe temporary decoding.
- Expanded forensic watermark export support beyond MP3 and added SoundCloud-specific forensic export behavior with SoundCloud fixed as the recipient and public SoundCloud profile metadata included where safe.
- Added tests around forensic export services, dialogs, controller paths, authenticity verification, conversion behavior, and SoundCloud token persistence.

## Accounting and Business Workflow Qualification
- Added invoice workspace package coverage and qualification assets for invoice issue, line entry, posting, payment entry, credit notes, royalty statements, payouts, and report generation.
- Extended accounting and royalty workflow qualification so UI-driven actions, database state, ledger state, generated reports, and evidence screenshots are checked together.
- Added UI reachability improvements including direct double-click navigation from relevant tables to the underlying record where applicable.

## Internal and Build Changes
- Synced local release work on top of the online `main` branch, including the v4.0.1 through v4.0.5 release metadata and workflow fixes.
- Updated release automation documentation and package metadata for the v5.0.0 major release.
- Added dedicated tests for crash detection, sanitisation, pending storage, rate limiting, secure proxy submission, GitHub issue form/label configuration, and report proxy validation.
- Preserved the no-bundled-token model: the desktop app never contains GitHub write credentials.

## Compatibility Notes
- This release remains Python 3.14.4-only.
- Direct GitHub issue creation from the desktop app is intentionally not supported with bundled credentials. Configure `ISRC_REPORT_PROXY_URL` and a server-side GitHub App proxy for online submission.
- If the proxy is unavailable or rejected by policy, users can still copy the preview or use the saved pending Markdown report.
