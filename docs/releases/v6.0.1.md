# ISRC Catalog Manager 6.0.1

[![Release](https://img.shields.io/github/v/tag/cosmowyn/ISRC-Catalog-Manager?label=release)](https://github.com/cosmowyn/ISRC-Catalog-Manager/releases/tag/v6.0.1)

Version: 6.0.1
Date: 2026-06-03
Type of update: Patch

## Fixes
- Fixed packaged update version reporting so frozen builds identify themselves with the release version after an automatic update.
- Added release smoke validation that fails the publishing workflow when the packaged binary reports a different version than the release manifest.
- Added source-version regression coverage so `pyproject.toml` and the frozen-build fallback in `isrc_manager/version.py` cannot drift.

## Update Notes
- This release supersedes v6.0.0 for automatic updates. The v6.0.0 package installed successfully, but its packaged smoke output and runtime fallback still reported v5.1.0.
- Users updating from v5.1.0 should receive v6.0.1 from the GitHub Release manifest.
- Users who already installed v6.0.0 should also be offered v6.0.1 because their running app reports v5.1.0 from the stale frozen fallback.

## Validation Focus
- Release QA now checks the packaged smoke output against `dist/release_manifest.json`.
- The release-build workflow still verifies compile, Ruff, Black, mypy, release automation tests, build requirements tests, sync-version tests, Python 3.14.4 compatibility, package validation, and packaged binary smoke tests before publishing release assets.
