# v6.0.3 Encrypted Profile Switch Fix Handoff

## Summary
- Investigated a remaining crash during profile switching after the encrypted database password prompt.
- Released the correction as v6.0.3 so v6.0.2 installations receive the safer profile-switch sequence through the normal updater.

## Root Cause
- The profile switch path closes the current database before opening the selected encrypted profile.
- During close, profile-scoped services were cleared and then visible workspace docks were refreshed immediately.
- Any visible dock whose refresh expected a live profile service could raise during teardown, after the password prompt had already succeeded.
- The async activation tail also used tuple callbacks, so late finalization failures could escape Qt callback handling instead of being logged and reported as profile activation failures.

## Changes
- Removed teardown-time workspace dock refresh from `_close_database_connection`; the new profile refreshes docks after foreground services are initialized.
- Replaced profile-switch tuple callbacks with named guarded callbacks.
- Added activation-failure logging/reporting around the post-prepare profile activation tail.
- Updated Help profile workflow text for encrypted profile switching.
- Added app-shell regression coverage for unlocking and switching to an encrypted profile through the password prompt.
