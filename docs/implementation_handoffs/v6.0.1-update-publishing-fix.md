# v6.0.1 Update Publishing Fix Handoff

## Summary
- Investigated GitHub issue 21, where automatic update installation detected v6.0.0 but the restarted app still reported v5.1.0.
- Confirmed the GitHub Release manifest and Windows asset pointed to v6.0.0, but the v6.0.0 tag still had `isrc_manager/version.py` fallback `__version__ = "5.1.0"`.
- Confirmed the release build warning was non-fatal GitHub Actions/cache noise; the actionable workflow gap was that packaged smoke printed `5.1.0` without failing the release job.

## Root Cause
- Frozen builds can fall back to `isrc_manager.version.__version__` when package metadata is unavailable inside the PyInstaller artifact.
- v6.0.0 had `pyproject.toml` set to `6.0.0`, but `isrc_manager/version.py` still reported `5.1.0`.
- The existing packaged smoke test only asserted that the binary launched and printed a smoke success message; it did not compare the reported version to `dist/release_manifest.json`.

## Changes
- Bumped the corrective release to v6.0.1.
- Synced `isrc_manager/version.py` with `pyproject.toml`.
- Added release smoke validation that rejects packaged binaries whose smoke output version does not match the release manifest.
- Added regression coverage requiring the runtime fallback version to match project metadata.
- Updated release notes and release-build documentation for the new validation gate.

## Expected Release Flow
1. Push the v6.0.1 commit to `main`.
2. Tag the same commit as `v6.0.1`.
3. The `Release Builds` workflow checks out `v6.0.1`, builds all platform packages, smoke-tests each package, verifies the smoke-reported version, prepares release notes/checksums/latest.json, and publishes the GitHub Release.
4. Existing v5.1.0 or v6.0.0 installations should detect v6.0.1 through `releases/latest/download/latest.json`.

## Validation
- Run focused version/update tests before publishing.
- Run release QA tests used by `.github/workflows/release-build.yml`.
- Confirm the online release build logs show packaged smoke output for v6.0.1, not v5.1.0.
